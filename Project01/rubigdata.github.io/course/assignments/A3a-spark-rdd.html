Assignment 3 Part A Hands-on session with Spark RDDs

Spark RDDs

The goal of assignment 3A is to get hands-on experience in using the Spark Notebook to write your spark programs.

Start a new docker container following the Spark Notebook background instructions.

You can access the Spark Notebook environment in your browser by navigating to localhost:9000. (Feel free to take a quick peek, but do not spend too much time before following the instructions below.)

Prepare container

For the assignment, we copy the _course notebooks prepared for the assignments into the directory with notebooks on the container.

Assignment A3a consists of two Spark notebooks, where the first notebook serves to introduce Spark and the second notebook takes a deep dive in the query processing mechanisms of Spark:

  1. BigData-big-data-spark-rdd-part-1.snb.ipynb
  2. BigData-big-data-spark-rdd-part-2.snb.ipynb

You can preview these Spark notebooks in your browser as (imperfectly) rendered by Github UI.

To get these notebooks onto your rubigdata/hadoop docker container, the recommended approach is to copy them from your assignment three repository hey-spark into a new directory on the container, using docker cp:

docker exec snb mkdir /opt/docker/notebooks/BigData
docker cp BigData-big-data-spark-rdd-part-1.snb.ipynb snb:/opt/docker/notebooks/BigData

Using Notebook

Navigate to localhost:9000/tree/BigData in your browser and open the notebook you just copied into inside your Spark Notebook container.

Navigate the notebook using the keyboard, pressing shift-enter to execute a cell, and enter to add lines to a cell. You can try out any scala command - create a new cell, enter some scala code in the cell, and execute it, just to get familiar with the notebook environment.

You will have to copy the Shakespeare file into the notebook, as last week, and/or modify its path in the notebook. For example:

docker exec snb mkdir /mnt/bigdata
docker cp 100.txt snb:/mnt/bigdata

This is just an example, the exercise in the notebook expects the 100.txt to be in the folder /opt/docker/hadoop-2.9.2/100.txt! Follow the steps in the course notebook to get at ease with using spark, scala; and make sure that you understand what you find in the Spark UI, available at localhost:4040 (after starting a kernel by opening the Spark Notebook).
